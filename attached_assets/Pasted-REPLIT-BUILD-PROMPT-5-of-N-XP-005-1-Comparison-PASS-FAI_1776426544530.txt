PART A — Load and transform the digitised κ_obs map

The file data/digitised/clowe2006_kappa.csv contains contour traces from 
Clowe et al. 2006 Figure 1 (weak-lensing convergence map). 

CSV structure:
- Header row 1: contour names in pairs (kappa_0.16,,kappa_0.23_main,,...)
- Header row 2: X,Y,X,Y,...
- Data rows: coordinate pairs per contour, varying lengths (empty cells 
  where shorter contours end)

The coordinates are in WebPlotDigitizer's internal system, NOT RA/Dec.
We must transform them using the calibration mapping.

STEP 1: Coordinate transformation

Four calibration anchor points were placed on the figure:

  X1_dig = -5.6078e-3  → RA = 104.675°  (6h58m42s, left edge)
  X2_dig = -4.9944     → RA = 104.550°  (6h58m12s, right edge)
  Y1_dig = -2.8039e-2  → Dec = -55.9667° (-55°58', bottom)
  Y2_dig = -5.5508e-2  → Dec = -55.9333° (-55°56', top)

Linear transform for each digitised point:

  RA  = 104.675 + (X_dig - X1_dig) * (104.550 - 104.675) / (X2_dig - X1_dig)
  Dec = -55.9667 + (Y_dig - Y1_dig) * (-55.9333 - (-55.9667)) / (Y2_dig - Y1_dig)

Verify by checking:
  - Main cluster contours should centre near RA ≈ 104.63°, Dec ≈ -55.945°
  - Bullet contours should centre near RA ≈ 104.58°, Dec ≈ -55.945°
  Print the centroid (mean RA, Dec) of each contour to confirm.

STEP 2: Transform RA/Dec to grid pixel coordinates

Our grid (from grid_params.json created in Module 1):
  - 200 x 200 pixels, 2 arcsec/pixel
  - Grid centre pixel (100, 100) = cluster centre

Load grid_params.json to get the grid centre RA/Dec. If not stored there, 
use RA_center = 104.625°, Dec_center = -55.950° (standard Bullet Cluster 
centre, midpoint of the Clowe figure).

  col = 100 + (RA_center - RA) * 3600 * cos(Dec_center_rad) / pixel_scale
  row = 100 - (Dec - Dec_center) * 3600 / pixel_scale

  (col increases eastward = decreasing RA;
   row increases downward in array = decreasing Dec;
   row 0 = north, row 199 = south)

Print: for each contour, how many points fall inside the 200x200 grid 
vs outside. Some outer contour points may fall outside — that's expected.

STEP 3: Convert contour lines to a filled 2D κ_obs map

The digitised data gives us points ALONG contour lines, not a filled map.
We need to reconstruct the filled map:

  a) Parse the CSV to extract each contour as a list of (col, row) points
  b) Close each contour polygon (connect last point to first point)
  c) For each contour level, use matplotlib.path.Path to determine which 
     grid pixels are inside the polygon
  d) Build κ_obs by assigning each pixel the HIGHEST contour value that 
     contains it:
     
     kappa_obs = np.zeros((200, 200))
     
     Process contours from LOWEST to HIGHEST kappa value:
       0.16 → fill interior with 0.16
       0.23 → overwrite interior with 0.23
       0.30 → overwrite interior with 0.30
       0.37 → overwrite interior with 0.37
     
     For contours with suffixes (kappa_0.23_main, kappa_0.23_bullet), 
     treat them as the SAME kappa level — they're separate spatial regions 
     at the same convergence value.
     
     Process order: 0.16 (all), 0.23 (all), 0.30 (all), 0.37 (all)

  e) Normalise: kappa_obs_norm = kappa_obs / max(kappa_obs)
     (max should be 0.37, so normalised peak = 1.0)
     Also keep the unnormalised version for the report.

  f) Handle the case where contours aren't perfectly closed: 
     if first and last points of a contour are more than 5 pixels apart, 
     print a warning but still close the polygon.

  g) Save: data/processed/kappa_obs.npy (normalised)
     Save: data/processed/kappa_obs_unnorm.npy (physical κ values)

  - Shape, min, max (normalised and unnormalised)

PART B — Run comparison statistics

Load from data/processed/:
  - kappa_model.npy
  - kappa_baseline.npy 
  - kappa_obs.npy (normalised)

All three maps should be 200x200 and normalised to [0,1].

COMPARISON REGION: Only compare pixels where kappa_obs > 0.
Pixels outside all contours have κ_obs = 0 because we have no data there 
(not because κ is actually zero). Comparing against these would inflate 
agreement artificially. Create a boolean mask:
  mask = kappa_obs > 0

Print: number of pixels in comparison region.

METRIC 1 — χ²/dof improvement

  Use uniform weighting (no per-pixel σ_obs available — record this in output):
  
  sigma_uniform = 0.14  
  (Bradač et al. 2006 reports ~14% measurement uncertainty over full ACS field)

  chi2_model = np.sum(((kappa_model[mask] - kappa_obs[mask]) / sigma_uniform)**2)
  chi2_baseline = np.sum(((kappa_baseline[mask] - kappa_obs[mask]) / sigma_uniform)**2)
  
  dof = np.sum(mask)  # number of comparison pixels, zero free parameters
  
  chi2_dof_model = chi2_model / dof
  chi2_dof_baseline = chi2_baseline / dof
  
  improvement = (chi2_dof_baseline - chi2_dof_model) / chi2_dof_baseline * 100
  
  PASS if improvement >= 25%

METRIC 2 — Peak offset

  Find pixel location of max(kappa_model) and max(kappa_obs).
  Convert pixel separation to arcsec: offset = distance_pixels * pixel_scale
  
  PASS if offset < 30 arcsec

METRIC 3 — Residual RMS

  residuals = kappa_model[mask] - kappa_obs[mask]
  rms = np.sqrt(np.mean(residuals**2))
  
  PASS if rms <= 0.2

METRIC 4 — Stability across galaxy subsamples

  This requires re-running Modules 2–3 with subsampled galaxy catalogues.
  
  For i in [1, 2, 3]:
    a) Randomly sample 2/3 of galaxies (use seed = 42 + i for reproducibility)
    b) Rebuild chi(x,y) from subsample
    c) Recompute P, psi_emit, psi_prop, psi_obs, kappa_model_sub
    d) Recompute chi2_dof_sub against kappa_obs
    e) Store chi2_dof_sub
  
  variation = (max(chi2_dof_values) - min(chi2_dof_values)) / mean(chi2_dof_values) * 100
  
  PASS if variation < 10%

  Note: import the field construction and model functions rather than 
  duplicating code. Load rho, T, grad_T, SB from saved .npy files — 
  only chi changes per subsample.
